# 全国高校录取数据爬虫

高考录取数据爬取与导出工具，支持院校级和专业级分数线数据。

## 📊 数据来源

- **专业级详细数据**：mnzy.gaokao.cn（广东省模拟志愿填报系统）
- **院校级汇总数据**：static-data.gaokao.cn（全国 31 省）

## 🎯 已生成成果

### 1. 广东省专业级详细数据

**文件**：`全国高校广东录取数据.md` / `.xlsx`（7.3 MB / 6.1 MB）

**内容**：
- **1717 所院校**（本科 1048 所 / 专科 669 所）
- **33681 条专业记录**（2023–2025 年）

**格式**：每所大学独立章节，按专业组列表展示，含锚点目录跳转。

## ⚠️ 数据约束说明

### 1. 专业级数据仅限广东

**原因**：mnzy API 的 `universityAnalysis` 端点只覆盖广东省（其他省份返回空数据）。

**影响**：无法生成其他省份的"专业 + 最低分/位次/**录取数**"详细表。

### 2. 院校级数据缺 8 省

**原因**：当前脚本 `export_prov.py` / `gen.py` 仅提取 `pro_type_min` 中的物理类（2073）和历史类（2074）数据，北京/天津/上海/浙江/山东/海南为 3+3 模式（综合改革，type=3），未单独处理。

**解决**：需修改脚本增加对 type=3 的支持。

### 3. 专业级爬虫已失效

**原因**：`crawl2.py` 依赖的 `static-data.gaokao.cn/www/2.0/schoolprovinceindex/...` 端点返回 404（OSS NoSuchKey）。

**影响**：无法通过该接口获取各省专业级分数。

## 📝 注意事项

1. **数据时效性**：当前数据覆盖 2023–2025 年，需定期更新爬虫适配新年份。
2. **API 稳定性**：mnzy 和 static-data 接口未提供官方文档，可能随时变更。
3. **签名校验**：mnzy API 的 `signsafe` 参数当前不做校验（任意值均可通过）。
4. **编码问题**：所有脚本顶部已设置 `sys.stdout = io.TextIOWrapper(..., encoding='utf-8')`，确保中文正常输出。
